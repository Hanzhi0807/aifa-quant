# AifaQuant 产品迭代路线图

> 本文档基于与 [AKQuant](https://github.com/akfamily/akquant) 的对比分析，确定 AifaQuant 的差异化定位与分阶段迭代方向。

---

## 1. 项目定位

**AifaQuant 不是“另一个 AKQuant”。**

AKQuant 是一个面向量化开发者的高性能 SDK（Rust 核心 + AKShare 生态），提供底层工具（表达式引擎、技术指标、实盘网关），但**不解决“选哪只股票”**的问题。

AifaQuant 的定位是：

> **面向“懂投资但不想写代码”的 A股投资者的 AI 选股产品。**

用户打开浏览器就能看到：
- 今天 AI 推荐买哪几只股票；
- 为什么推荐这些股票（因子贡献、模型置信度）；
- 模拟盘历史表现如何；
- 每个因子是否真正有效（IC、分层收益）。

**核心壁垒是“选股有效性”，不是“交易执行速度”。**

---

## 2. 与 AKQuant 的对比结论

| 维度 | AKQuant | AifaQuant | 判断 |
|------|---------|-----------|------|
| 选股框架完整度 | 只有空壳适配器 | 端到端 pipeline（数据 → 因子 → 模型 → 策略 → 回测 → 模拟交易） | ✅ 领先 |
| 因子丰富度 | 103 个 TA 指标 + Alpha101 表达式引擎 | ~20 个手工因子 | ⚠️ 偏弱 |
| 模型多样性 | 用户自己接 | 仅 LightGBM 二分类 | ⚠️ 偏窄 |
| 截面 Alpha 因子 | Polars 表达式引擎 | 无 | ⚠️ 差距大 |
| 基本面 + 宏观因子 | 不涉及 | 已内置 PE/PB/ROE + CPI/PMI/M2 | ✅ 领先 |
| 因子有效性验证 | 无 | 无 | ⚠️ 都没有 |
| Web 产品化 | 无 | 已有 React Dashboard（待接入真实数据） | ✅ 领先 |
| 实盘交易 | CTP 网关 | 仅模拟交易（ intentionally ） | — 战略差异 |

**结论**：我们的差距在**因子深度、模型多样性、因子有效性证明**，不在框架完整度。不要和 AKQuant 拼底层基础设施，要拼“用户能看懂的选股结果”。

## 3. 迭代路线

### Phase 1 — 产品可用 + 因子有效性（2-3 周）

目标：让用户能零配置跑起来，并在 Web 上看到 AI 选股结果和证据。

- [ ] **因子有效性分析模块**
  - IC / RankIC / ICIR 计算
  - 分层回测（Quantile Portfolio）及可视化
  - 因子衰减曲线（未来 1/5/10/20 天 IC）
  - 输出位置：`aifa_quant/analysis/factor_analysis.py`

- [ ] **Web Dashboard 接真实数据**
  - 后端从 DuckDB 读取 `daily_quotes` / `paper_positions` / `paper_orders` / `paper_nav`
  - 前端展示：今日选股信号、当前持仓、净值曲线、因子有效性图表
  - 移除/弱化对 MySQL 的强依赖，提供 SQLite/DuckDB 默认模式

- [ ] **零配置启动**
  - `docker compose up` 一键启动（含 DuckDB、可选 Release 数据自动下载）
  - 默认不需要 iFind token；日线/指数/成分股走 AkShare
  - 首次启动自动导入 Release 数据或提示下载

- [ ] **模拟盘可视化**
  - 在 Web 展示“今日 AI 推荐”与“历史模拟盘盈亏”
  - 订单明细、持仓成本、每日净值

- [ ] **交付物**
  - 一个 Docker 一键启动的 Web 产品
  - 用户无需 iFind token 即可看到 AI 选股结果
  - 每个推荐股票附带“为什么选它”的因子贡献摘要

### Phase 2 — 因子库扩展 + 模型升级（3-4 周）

目标：显著提升选股有效性，建立模型解释能力。

- [ ] **因子库扩展**
  - 接入 Alpha101 / Alpha191 截面因子（用 pandas/pandas-ta 实现，不自研表达式引擎）
  - 行业因子、市值中性化、流动性因子、波动率因子
  - 因子正交化 / 去市值暴露
  - 目标：50-100 个经过 IC 验证的有效因子

- [ ] **模型升级**
  - 从二分类改为排序问题：LambdaRank / LightGBM Ranker
  - 多模型 ensemble：LightGBM + XGBoost + CatBoost
  - 自动特征选择：递归消除 / Boruta
  - 模型置信度：predict_proba 分位数 → “把握有多大”

- [ ] **可解释性**
  - SHAP 分析集成到 Web
  - 每只股票展示“Top 5 贡献因子”
  - 全局特征重要性趋势图

- [ ] **交互式因子商店**
  - Web 上勾选/关闭因子组合
  - 实时重新跑回测并对比净值曲线
  - 保存为自定义策略模板

- [ ] **舆情因子（免费源）**
  - 接入东方财富 / 新浪新闻
  - 简单 NLP 情绪得分（正面/负面/中性）
  - 作为可选因子，不依赖 iFind news MCP

### Phase 3 — 增长与社区（4-6 周）

目标：降低使用门槛，建立内容获客能力。

- [ ] **PyPI 发布**
  - `pip install aifa-quant`
  - 包含 CLI 和 Web 启动命令

- [ ] **策略模板市场**
  - 内置 5-10 个选股策略模板：动量、价值、质量、低波、多因子混合
  - 用户一键回测、一键模拟交易

- [ ] **文档站**
  - MkDocs + GitHub Pages
  - 教程风格："如何用 AI 选股"，而非纯 API 文档

- [ ] **每周 AI 选股报告**
  - GitHub Actions 定时跑模型（若 Actions 恢复）
  - 自动生成并发布到 GitHub Discussions / 公众号 / 文档站
  - 包含 Top-K 选股、模型置信度、上周表现

- [ ] **免费数据源完善**
  - ✅ AkShare 已接入（日线/指数/成分股）
  - 可选 Tushare 适配器（需要用户 token）
  - 基本面数据若 AkShare 稳定，逐步迁移部分字段

## 4. 红线（不要做）

- **不要用 Rust 重写引擎**：Python 足够，用户不感知回测速度差异。
- **不要支持期货/期权/加密货币**：专注 A股选股。
- **不要做实盘交易接口**：法律风险高，且目标用户群不需要。
- **不要在交易执行层花大量时间**：模拟交易已够用，壁垒在选股。
- **不要追求 star 数量**：追求“有人每天打开 Dashboard 看选股信号”。
- **不要默认开启 iFind**：任何 iFind 调用都必须用户显式确认（已实施 `--yes`）。

---

## 5. 下一步建议（从 Phase 1 开始）

按优先级排序，建议先做以下 3 件事：

1. **因子有效性模块**（2-3 天）
   - 实现 IC/RankIC/分层回测
   - 为现有 ~20 个因子出具第一份有效性报告
   - 这能直接指导 Phase 2 该扩哪些因子

2. **Web 后端接 DuckDB**（3-5 天）
   - 让前端能展示真实选股结果和模拟盘状态
   - 这是产品化的关键瓶颈

3. **Docker 一键启动**（2-3 天）
   - 把 Web + DuckDB + 自动数据导入打包
   - 让非技术用户能跑起来

完成这三项后，AifaQuant 就从“开发框架”变成“可演示的 AI 选股产品”。
