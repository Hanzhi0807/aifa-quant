## 1. 项目概述

AifaQuant 是一个面向 A股市场的个人 AI 量化研究与回测框架。本期完成了从数据接入到回测验证的完整闭环。

## 2. 已完成工作

### 2.1 项目骨架
- 创建了模块化 Python 项目结构
- 配置了 `.env` 环境变量管理（token 安全隔离）
- 安装了核心依赖：pandas, duckdb, lightgbm, backtrader, typer 等

### 2.2 数据层
- 实现 iFind MCP Streamable HTTP 客户端
- 支持 `tools/list` 自动发现 和 `tools/call` 调用
- 适配 iFind 自然语言查询返回的 Markdown 表格
- 标准化列名、解析中文单位（万/亿）
- 接入 DuckDB 本地存储，支持增量更新

### 2.3 因子工程
- 技术面因子：收益率、移动均线、RSI、MACD、波动率、ATR、成交量
- 标签：未来 N 日收益率 + 二分类涨跌标签
- 自动剔除高缺失率特征、中位数填充缺失值
- 已排除未来收益类特征，避免数据泄露

### 2.4 模型
- LightGBM 二分类选股模型
- 模型保存/加载机制
- 特征重要性输出

### 2.5 策略与回测
- TopK-Dropout 轮动策略
- 自定义 A股回测引擎（考虑 T+1、涨跌停、100 股整数手、手续费、印花税）
- 绩效指标：总收益、年化收益、夏普、最大回撤、日胜率
- 权益曲线 CSV + PNG 输出

## 3. 运行结果

- 时间范围：2023-01-03 至 2024-12-31

## 4. 已知问题与后续方向

### 4.1 当前局限
1. **样本规模小**：仅 10 只股票、2 年数据，模型容易过拟合。
2. **数据范围**：iFind 返回的日线数据每次约 100 条，需要更长时间序列需分多次拉取或调整查询。
3. **A股规则简化**：涨跌停判断使用成本价替代昨日收盘价，实际应维护完整历史前收盘价。
4. **未做交叉验证**：训练集/测试集按年份划分，未做滚动验证。

### 4.2 下一步建议
1. 扩展股票池至 50-300 只，增加样本多样性
2. 加入财务、估值、宏观、新闻情绪等更多因子
3. 实现滚动训练/验证，降低过拟合
4. 接入更多基准（如沪深 300）进行超额收益分析
5. 增加参数优化和策略对比
6. 接入模拟盘/实盘接口（QMT/easytrader）
